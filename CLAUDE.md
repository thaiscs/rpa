## Project Overview

RPA (Robotic Process Automation) system for managing digital certificates. Architecture consists of:

- **api/**: FastAPI backend with JWT authentication (fastapi-users) and certificate management
- **ui/**: NiceGUI frontend for admin panel
- **worker/**: Background worker (currently disabled in docker-compose)
- **shared/**: Shared utilities (database models, crypto, CRUD operations)
- **alembic/**: Database migrations

## Common Commands

### Development
```bash
# Run all services
docker compose up --build

# Stop and clean
docker compose down --remove-orphans
docker compose down -v  # delete volumes

# View logs
docker compose logs -f api
docker compose logs -f ui

# Database access
docker exec -it rpa-postgres-1 psql -U postgres -d certsdb
```

### Migrations
```bash
# Generate migration
alembic revision --autogenerate -m "description"

# Run migrations (via docker compose)
docker compose run migrations

# Rollback
alembic downgrade -1

# View history
alembic history --verbose
```

### Local Development (without Docker)
```bash
# API
cd api && python main.py

# UI
cd ui && python main.py
```

### Testing
```bash
# Install test dependencies
pip install -r requirements-test.txt

# Run all tests
pytest

# Run with coverage
pytest --cov=. --cov-report=html

# Run specific module
pytest tests/shared/test_crypto.py

# Run with verbose output
pytest -v
```

## Architecture Notes

### Authentication
- Uses `fastapi-users` with JWT backend
- Session lifetime: 8 hours
- Auth secret loaded from `/secrets/auth.key` (generated by init-secrets service)
- Two user roles: `current_admin` (authenticated) and `current_superuser` (admin only)

### Database
- PostgreSQL 16 with async SQLAlchemy
- Models in `shared/shared/models/`: User, Client, Certificate
- User extends `SQLAlchemyBaseUserTableUUID` with optional `name` and `client_id`

### Certificate Encryption
- Fernet encryption for all certificate data (PFX, password, cert, key)
- Key loaded from `/secrets/fernet.key` (generated by init-secrets service)
- Encrypted data stored as JSONB with version field
- `shared/shared/crypto.py` handles encryption/decryption and PFX parsing

### API Routes
- `/auth/*`: Authentication (login, signup, password reset, verification)
- `/users/*`: User management
- `/admin/*`: Admin routes (upload-cert, send-job) - requires authentication
- Public routes in `api/routes.py` (currently minimal)

### UI Structure
- Pages in `ui/pages/`: login, signup, dashboard, clients, ecac, add_cert
- Components in `ui/components/`: sidebar, cert_form, error dialogs/toasts
- Helpers in `ui/helpers/`: auth, parsing, secret, validation
- Auth helper manages JWT token in `app.storage.user` with expiration check

### Service Dependencies
- `init-secrets`: Generates Fernet and auth keys, must complete before api
- `migrations`: Runs alembic migrations, must complete before api
- `postgres`: Database with healthcheck
- `rabbitmq`: Message queue (worker currently disabled)

### Key Files
- `api/main.py`: FastAPI app with route registration
- `api/auth/`: Authentication configuration and dependencies
- `shared/shared/crud.py`: Certificate CRUD operations with encryption
- `shared/shared/db.py`: Database connection and session management
- `alembic/env.py`: Migration configuration (imports all models)

## Test Structure

Tests are organized by module in `tests/` directory:
- `tests/shared/`: Tests for shared utilities (crypto, utils, crud, db, models)
- `tests/api/`: Tests for API module (auth, admin routes, main app)
- `tests/ui/`: Tests for UI helpers (auth, validation, secret)
- `tests/scripts/`: Tests for scripts (init-secrets)
- `tests/conftest.py`: Shared fixtures for all tests

Tests use pytest with async support, mocking for external services, and separate test database. Worker module tests are excluded.
